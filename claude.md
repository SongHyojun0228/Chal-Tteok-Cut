# 🪮 찰떡컷 - 프로젝트 개요

> AI 기반 헤어스타일 추천 앱  
> 두상·얼굴형·모질 분석으로 나에게 딱 맞는 스타일을 찾아드립니다

---

## 📌 프로젝트 요약

### 프로젝트명
**찰떡컷** (Chaltteok-Cut)

### 한 줄 소개
"찰떡같이 어울리는 헤어컷" - 두상·얼굴형·모질 분석으로 최적의 헤어스타일을 AI가 추천해주는 모바일 앱

### 핵심 가치
- **고객**: 미용실 가기 전 나에게 어울리는 스타일 미리 확인, 실패 확률 감소
- **미용사**: 상담 시간 30% 단축, 고객 만족도 향상, 재방문율 증가

### 개발 기간
**3개월 (MVP)**

### 팀 구성
- 개발자 1명 + AI 파트너 (Claude)
- **모토**: "바이브코딩으로 최고의 앱 만들자!" 🔥

---

## 🎯 타겟 사용자

### Primary User
- 20-40대 헤어스타일 관심층
- 새로운 스타일 시도하고 싶지만 실패 두려운 사람
- "어떤 스타일이 어울릴까?" 고민하는 사람
- 미용실 가서 "아무거나요" 하는 사람

### Secondary User
- 미용사 (고객 상담 효율화 도구)
- 미용실 (고객 만족도 & 재방문율 향상)

## 💡 핵심 기능 (MVP)

### 1. 사용자 프로필 생성

#### A. 얼굴 사진 촬영
- **필수**: 정면 1장
- **선택**: 측면(좌/우), 상단 (더 정확한 추천)
- 촬영 가이드:
  - 얼굴 가이드라인 오버레이 제공
  - 조명/각도 자동 체크
  - 실시간 피드백

#### B. 간단한 질문 (6개)
1. 모질: 직모 / 약한 웨이브 / 강한 곱슬
2. 모량: 적음 / 보통 / 많음
3. 두피 타입: 지성 / 중성 / 건성
4. 현재 머리 길이: 초단발 / 단발 / 중단발 / 긴머리
5. 관리 시간: 5분 이하 / 10분 / 20분 이상
6. 선호 스타일: 자연스러운 / 트렌디한 / 개성있는

#### C. 입력 플로우
총 소요시간: **3분 이내**
1. 환영 화면 (30초)
2. 사진 촬영 (1분)
3. 질문 응답 (1분)
4. AI 분석 중... (10초)
5. 추천 결과!

---

### 2. AI 분석 엔진

#### A. 얼굴형 자동 인식
**기술 스택**:
- MediaPipe Face Mesh (468개 랜드마크)
- 또는 Google Cloud Vision API

**분류 카테고리 (5가지)**:
1. 둥근형 (Round)
2. 계란형 (Oval) 
3. 각진형 (Square)
4. 긴형 (Oblong)
5. 역삼각형 (Heart)

**측정 지표**:
- 얼굴 길이 대 너비 비율
- 턱선 각도
- 이마 너비

#### B. 두상 형태 분석 (선택 사진 있을 경우)
- 뒷통수 볼륨
- 측면 윤곽
- 정수리 형태
- → 더 정밀한 추천에 활용

#### C. 추천 알고리즘

**Phase 1 (MVP): Rule-based**
```
IF 얼굴형 == "둥근형" AND 모질 == "직모"
  → 레이어드 긴머리, 앞머리 없는 중단발 추천
  
IF 얼굴형 == "각진형" AND 모량 == "많음"
  → 웨이브 펌, 부드러운 레이어 추천
```

**Phase 2: Collaborative Filtering**
- 유저 피드백 데이터 축적
- "비슷한 조건의 사람들이 만족한 스타일"
- TensorFlow Lite로 온디바이스 추론

---

### 3. 스타일 추천 결과

#### A. 추천 결과 화면
각 스타일 카드에 표시:
- ⭐ 매칭 점수 (%)
- 스타일명 & 대표 이미지
- 추천 이유 (1-2줄)
- 관리 난이도 (★☆☆ ~ ★★★)
- 예상 시술 시간
- 예상 가격대

#### B. 각 스타일 카드 정보
- 스타일 이름
- 대표 이미지 (3-4장)
- 매칭 점수 (%)
- 추천 이유
- 관리 난이도
- 예상 시술 시간
- 예상 가격대
- 어울리는 옷 스타일 (보너스)

#### C. 상세 페이지
- 스타일 설명
- 시술 과정 간단 설명
- 관리 방법 (홈케어 팁)
- 유지 기간
- 비슷한 스타일 추천
- "이 스타일 저장" 버튼
- "미용사와 공유" 버튼

---

### 4. 스타일 시뮬레이션 (Phase 1.5)

**옵션 A**: 간단한 AR 오버레이
- 스타일별 헤어 오버레이 이미지
- 사용자 얼굴에 합성
- 실시간 프리뷰

**옵션 B**: 유사 사례 매칭
- "당신과 비슷한 얼굴형의 Before/After"
- 실제 시술 사례 이미지 제공

---

### 5. 미용사 연동 (Phase 2)

#### A. 프로필 공유 기능
1. 고객이 선호 스타일 선택
2. QR코드 또는 링크 생성
3. 미용실 방문 시 미용사에게 제시
4. 미용사는 앱/웹에서 프로필 확인
   - 얼굴형, 모질, 모량
   - 선호 스타일
   - 관리 가능 시간

#### B. 미용사 피드백 시스템
- 시술 후 미용사가 "실제로 한 스타일" 기록
- "추천이 적절했나요?" 평가
- 피드백 데이터로 추천 알고리즘 개선

### 상세 스택

#### Firebase
**역할**: 
- 사용자 인증/관리
- 간단한 데이터 저장 (프로필, 설정)
- 이미지 저장
- 실시간 동기화

**선택 이유**:
- ✅ 개발 속도 빠름
- ✅ 인프라 관리 Zero
- ✅ 무료 티어로 MVP 가능
- ✅ Auto-scaling

**데이터 구조**:
```
users/
  {userId}/
    - profile: {face_shape, hair_type, ...}
    - preferences: {...}
    - saved_styles: [...]
    - history: [...]
```

---

#### Cloud Run (ML Backend)
**역할**:
- 얼굴형 인식 API
- 스타일 추천 엔진
- 무거운 이미지 처리

**선택 이유**:
- ✅ 서버리스 (사용한 만큼만 과금)
- ✅ Python ML 라이브러리 사용 가능
- ✅ Auto-scaling
- ✅ Cloud Functions보다 메모리/CPU 여유

**API 엔드포인트**:
```
POST /api/analyze-face
  - Input: 얼굴 이미지 (base64)
  - Output: {face_shape, confidence, landmarks}

POST /api/recommend-styles
  - Input: {face_shape, hair_type, preferences}
  - Output: [스타일 리스트]
```

---

#### PostgreSQL (Cloud SQL)
**역할**:
- 스타일 데이터베이스 (메인)
- 복잡한 쿼리 처리
- 미용사 데이터 (Phase 2)

**선택 이유**:
- ✅ 복잡한 JOIN, 필터링 가능
- ✅ Firestore보다 쿼리 유연
- ✅ 스타일 DB는 구조화된 데이터

**테이블 구조**:
```sql
styles
  - id
  - name
  - category
  - face_shapes[] (적합한 얼굴형들)
  - hair_types[] (적합한 모질)
  - hair_amounts[] (모량)
  - difficulty_level
  - maintenance_time
  - image_urls[]
  - description
  - price_range
```

| 펌 | 없음/웨이브/C컬/S컬 |

## 🎯 차별화 포인트

### vs. 기존 헤어스타일 앱
- ✅ 얼굴형뿐 아니라 모질, 두상까지 고려
- ✅ AI 기반 정교한 추천
- ✅ 미용사 연동 (실제 시술로 연결)

### vs. AR 시뮬레이션 앱
- ✅ 단순 재미가 아닌 실용적 추천
- ✅ 관리법, 난이도 정보 제공
- ✅ 미용실 방문 전 사전 상담 도구

### vs. 미용실 예약 앱
- ✅ 스타일 결정 단계에 집중
- ✅ 고객-미용사 커뮤니케이션 개선
